# v5.0 集成风险评估报告

**评估时间**: 2026-07-04  
**状态**: 记录 only，不写代码  
**决策基础**: quorvex_ai 深度评估 + v5.0 架构草案

---

## 1. 风险评估矩阵

### 1.1 quorvex_ai 直接 fork 风险

| 风险 | 概率 | 影响 | 缓解 |
|---|---|---|---|
| 架构不匹配 (browser vs MES) | 高 (~80%) | 高 — 需 >50% 重写 | ❌ 不直接 fork |
| 依赖冲突 | 中 (~40%) | 中 — 可能影响现有 pip 环境 | 环境变量 gate + 沙箱 |
| 安全漏洞 | 低 (~20%) | 高 — 权限升级路径不可控 | 硅基体已有消毒层隔离 |
| 维护成本 | 高 (~70%) | 中 — 需跟踪上游更新 | 仅记录不集成，无维护成本 |

### 1.2 v5.0 调度器新开发风险

| 风险 | 概率 | 影响 | 缓解 |
|---|---|---|---|
| 并行执行竞争条件 | 中 (40%) | 中 — 修复结果冲突 | 策略间状态隔离 + 互斥锁 |
| S3 权限预取被误触发 | 低 (20%) | 高 — 非预期权限提升 | 超时自动释放 + 审计日志 |
| 调度器 + 现有引擎兼容 | 低 (10%) | 低 — 不影响串行路径 | 调度器是新文件，不修改现有代码 |
| LLM fallback 引入外部依赖 | 中 (50%) | 中 — 需 API key | 默认 `ENABLE_LLM_FALLBACK=false` |

### 1.3 合规风险 (7.15 后)

| 风险 | 概率 | 缓解 |
|---|---|---|
| 调度器变更影响合规审计 | 低 | `v3.0-compliant` tag 已锁 |
| 新模块引入 bug 影响回滚 | 低 | 调度器不影响现有串行路径 |
| 观察窗数据污染 | 低 | 调度器运行时有独立日志 |

## 2. 依赖树 (proposed)

```
[用户请求]
    │
    ├── v5_scheduler.py (新) ← 仅此文件新增
    │   ├── apply_repair.py (不变)
    │   ├── mes_adapter.py (不变) ← --parallel 开关新增
    │   └── headroom_adapter.py (不变, 可选)
    │
    ├── fault_templates/*.yaml (增量扩展)
    └── scripts/*.py (运行脚本，不变)
```

**核心**:
- 只有 `v5_scheduler.py` + `mes_adapter.py --parallel` 需要变更
- `apply_repair.py` **零修改**（这是硬边界）

## 3. 回滚路径

| 回滚方案 | 操作 | 耗时 | 破坏性 |
|---|---|---|---|
| git revert | `git revert v5.0` | < 1 min | 无 |
| 环境变量 | `USE_PARALLEL=false` | 0 (运行时) | 无 |
| Tag checkout | `git checkout v4.0-compliant` | < 1 min | 无 |

## 4. 结论

| 决策 | 理由 |
|---|---|
| ❌ 不 fork quorvex_ai | 架构不匹配，不通过 |
| ✅ v5.0 调度器新开发可行 | 风险可控，增量变更 < 300 行 |
| ✅ 保持 gate 可选 | `USE_PARALLEL=false` 零伤切换 |
| ⚠️ LLM fallback 延后 | 默认 gated，v6.0 再评估 |
