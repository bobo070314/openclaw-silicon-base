**起草时间**: 2026-07-04

## 1. 目标

定义硅基体协议的 **v6.0 标准化字段集合**，确保：
- 新产业（锂电、光伏、储能、半导体）模板字段一致
- v10.0 协议标准化路径清晰
- 向下兼容 v2.0~v4.0 旧格式

## 2. v6.0 标准字段定义

### 2.1 必选字段 (MUST)

| 字段 | 类型 | 示例 | 说明 |
|---|---|---|---|
| `fault_id` | str | `F-042` | 全局唯一 ID |
| `name` | str | `sintering_temp_exceeded` | 机器可读名称 (snake_case) |
| `description` | str | `sintering temperature exceeds limit` | 人类可读描述 (英/中文均可) |
| `severity` | enum | `P1`, `P2`, `P3` | 严重等级，3 级制 |
| `category` | enum | `process_anomaly` | 故障分类 (见 2.3) |
| `repair_strategy` | enum | `S1`, `S2`, `S3` | 修复策略 |
| `signals` | list | `[temperature > 980°C]` | 可观测信号列表 (≥2) |
| `affects` | list | `[cathode_batch, crystal_grain]` | 受影响组件/系统 |

### 2.2 可选字段 (SHOULD)

| 字段 | 类型 | 说明 |
|---|---|---|
| `industry` | enum | 所属产业 (general/lithium/pv/storage/semicon) |
| `sensor_type` | str | 触发信号的传感器类型 |
| `threshold` | str | 触发阈值表达式 |
| `rbac_level` | enum | 修复所需权限等级 (L1/L2/L3) |

### 2.3 Category 枚举值

| Enum | 说明 |
|---|---|
| `process_anomaly` | 工艺参数偏差 |
| `equipment_fault` | 设备故障/停机 |
| `material_quality` | 原材料品质问题 |
| `quality_defect` | 产品缺陷 |

**(与当前 v4.0 category 完全一致)**

### 2.4 severity 与 rbac_level 映射

| Severity | RBAC Level | 修复时间窗 |
|---|---|---|
| P1 (critical) | L3 (system admin) | < 15 min |
| P2 (major) | L2 (line supervisor) | < 1 h |
| P3 (minor) | L1 (operator) | < 8 h |

## 3. 模板示例 (v6.0 新格式)

```yaml
# v6.0 标准格式示例
fault_id: F-042
name: energy_storage_bms_voltage_deviation
industry: storage
description: Battery Management System reports cell voltage deviation
severity: P1
category: process_anomaly
repair_strategy: S1
rbac_level: L2
sensor_type: bms_voltage
threshold: delta > 0.15V
signals:
  - cell_voltage_delta > 0.15V
  - soc_measurement_discrepancy > 5%
  - cell_count > 4
affects:
  - battery_pack
  - soc_accuracy
```

## 4. v2.0 兼容适配

v2.0 模板 (F-005~F-020) 格式与 v4.0 不同。自动补齐规则：

```python
# v6.0 兼容读取伪代码
def normalize_v2(fault_dict):
    """v2.0 单对象格式 → v6.0 标准格式"""
    fault_dict["name"] = fault_dict.get("name", fault_dict["fault_id"])
    fault_dict["category"] = fault_dict.get("category", "general")
    fault_dict["signals"] = fault_dict.get("signals", fault_dict.get("parameters", {}).get("signals", ["unknown"]))
    fault_dict["affects"] = fault_dict.get("affects", ["production_line"])
    return fault_dict
```

**无需物理迁移 v2.0 模板。** 运行时代码补齐即可。

## 6. 反馈机制 (optional)

v6.0 标准格式支持 `_meta` 字段用于反馈：

```yaml
fault_id: F-042
_meta:
  version: 6.0
  source_provider: hoshine_mes
  last_updated: 2026-07-04
  compatibility: ["mes_v2", "mes_v3"]
```
