# v6.0 协议标准化风险评估

**评估时间**: 2026-07-04  
**状态**: 记录 only，不写代码  
**数据基础**: v4.0 模板分析 + v6.0 协议草案

---

## 1. 风险矩阵

| ID | 风险 | 概率 | 影响 | 缓解 |
|---|---|---|---|---|
| R01 | v2.0 旧格式与新格式冲突 | 高 (70%) | 低 | 运行时代码补齐，不物理迁移 |
| R02 | 不同产业 category 命名不一致 | 中 (40%) | 中 | 统一 v6.0 枚举 (4 类，与 v4.0 一致) |
| R03 | 信号格式不统一 (混合字符串/数值) | 中 (50%) | 中 | 强制 `key operator value` 格式 |
| R04 | `affects` 字段内容形式不统一 | 中 (50%) | 中 | 文档规范，不强制语法检查 |
| R05 | v10.0 扩展导致字段膨胀 | 低 (30%) | 低 | `_meta` 字段隔离扩展 |

## 2. 各风险详细分析

### R01: v2.0 旧格式冲突

**实际验证结果**:
- v2.0 模板 (F-005~F-020): 缺 `name/category/signals/affects`
- v4.0 模板 (F-021~F-040): 缺 `fault_type/parameters/source`

**风险**: 新旧模板混用导致读取报错

**缓解方案** (无需改代码):
```python
def safe_read(fault_dict):
    if "templates" not in fault_dict:
        return normalize_v2(fault_dict)  # 自动补齐
    return fault_dict
```

**残留风险**: 🟢 低 (mes_adapter.py 已有 `yaml.safe_load`，不会因缺字段崩溃)

### R02: Category 命名不一致

**当前所有模板的实际 category 值**:

| 来源 | 使用的 category |
|---|---|
| F-021~F-030 (锂电) | `process_anomaly`, `material_quality`, `equipment_fault` |
| F-031~F-040 (光伏) | `process_anomaly`, `quality_defect`, `equipment_fault`, `material_quality` |
| F-005~F-020 (通用) | 缺 category 字段 |

**共识度**: 4 个 category 在锂电/光伏间完全一致。

**残留风险**: 🟢 低 (当前无命名冲突)

### R03: 信号格式不统一

当前示例：
- `temperature > 980°C` (数值 + 单位)
- `viscosity < 3000 mPa·s OR viscosity > 8000 mPa·s` (含逻辑)
- `line_status: emergency_stop` (键值对)
- `EL_test: micro_crack_detected` (枚举值)

**风险**: 深度修复引擎需解析信号。

**缓解**: 简单匹配用字符串 `in` 检查，无需解析数值。v10.0 再考虑结构化解析。

**残留风险**: 🟡 中 (不影响当前 S1/S2/S3 匹配)

### R04: Affects 格式不统一

当前示例：
- `[cathode_batch, crystal_grain_size]` (锂电)
- `[module_power, warranty_claim_risk]` (光伏)
- `[production_line, material_waste]` (光伏)

**问题**: 有些是物理设备名，有些是商业风险。

**缓解**: 只做文档规范，不做语法检查。不影响修复逻辑。

**残留风险**: 🟢 低

### R05: v10.0 字段膨胀

v6.0 有 8 个 MUST + 5 个 SHOULD = 13 个字段。  
v10.0 预计扩展到 18-20 个字段。

**缓解**: `_meta` 字段作为扩展舱，不污染核心字段。

## 3. 风险总评

| 等级 | 数量 | 描述 |
|---|---|---|
| 🟢 低 | 3 (R01, R02, R04) | 运行时兼容可解决 |
| 🟡 中 | 1 (R03) | 信号解析不影响当前匹配 |
| 🔴 高 | 0 | 无阻塞性风险 |

## 4. 结论

| 决策 | 理由 |
|---|---|
| ✅ v6.0 协议标准化可行 | 现有模板 21 条分析无阻塞性风险 |
| ✅ 向下兼容有方案 | `normalize_v2` 补齐函数 (运行时) |
| ⚠️ 信号解析延后 | 当前不必要，v10.0 再评估 |
| 📌 无需全量迁移 | 仅限于新建模板使用 v6.0 格式 |
