# v5.0 架构草案 — AI 自愈规则引擎

**起草时间**: 2026-07-04  
**状态**: Draft (仅文档，不写代码)  
**基础**: v2.0~v4.0 引擎 + quorvex_ai 深度评估

---

## 1. 核心变更 (vs v4.0)

| 变更 | 描述 | 优先级 |
|---|---|---|
| 并行策略执行 | S1/S2/S3 并行尝试，首胜返回 | P0 |
| 策略链回退 | S1 失败 → S2 → S3 → escalation | P1 |
| LLM fallback (可选) | 规则引擎未覆盖时调用 LLM | P2 |

**不变**:
- `apply_repair.py` 核心修复逻辑（S1/S2/S3）
- YAML 模板格式（`fault_id/name/signals/repair_strategy`）
- 输入消毒 + 路径隔离（安全层）
- `USE_HEADROOM` 开关（Headroom 集成）

## 2. 新组件: 策略调度器 (proposal)

### 2.1 接口

```python
class StrategyScheduler:
    """
    并行策略调度器。

    策略执行流:
        dispatch(fault_type, signals, context)
            → S1 启动 (async)
            → 同时预准备 S2 参数
            → S1 成功? → 返回结果
            → S1 失败? → S2 立即启动 (已预准备)
            → S2 失败? → 预取 S3 权限 → S3 启动
            → 全部失败? → escalate(fault)
    """
```

### 2.2 执行流

```
[输入] Fault + Signals
    │
    ├──→ [S1] 并行启动修复
    │   ├── 成功 → 返回
    │   └── 失败 → 触发 S2 (已有预准备)
    │
    ├──→ [S2 准备] 预计算配置补丁
    │   ├── 收到 S1 失败信号 → 立即执行
    │   └── 收到 S1 成功 → 丢弃
    │
    └──→ [S3 准备] 预获取 RBAC 权限
        ├── S2 也失败 → 执行 escalation
        └── 任意成功 → 释放权限
```

### 2.3 不修改现有模块

- `apply_repair.py` 保持为纯函数修复内核
- `scheduler.py` 作为外层编排层
- 通过 `mess_adapter.py --parallel` 开关触发

## 3. 配置变更

```yaml
# configs/v5_scheduler.yaml (proposal)
scheduler:
  parallel: true                    # 启用并行策略
  timeout_s: 15                     # 单策略超时
  chain_rollback: true              # 启用策略链回退
  llm_fallback: false               # 默认禁用 LLM
  llm_config:                       # 仅当 ENABLE_LLM_FALLBACK=true
    max_tokens: 500
    provider: openai
```

## 5. 风险评估

| 风险 | 概率 | 缓解 |
|---|---|---|
| 并行执行竞争条件 | 中 | 策略间共享状态加锁 |
| S3 权限预取浪费 | 低 | 超时自动释放 |
| LLM fallback 延迟 | 高 | 默认禁用，显式开启 |
| 调度器 + 现有引擎不兼容 | 低 | 调度器不影响现有串行路径 |

## 6. 建议实现顺序

```
v5.0.1: 调度器骨架 + 并行 S1/S2 实现 (P0)
v5.0.2: 策略链回退 + 预准备 (P1)
v5.0.3: LLM fallback 可选通道 (P2, gate 关闭)
v5.0.4: 观察窗 (7 days compressed)
v5.0.5: 封板 + tag
```

## 7. 结论

v5.0 不是重写引擎，而是**给现有 v2.0~v4.0 引擎加一层智能调度**。核心代码变更量 < 300 行（新文件 `scheduler.py`），最大风险来自 `apply_repair.py` —— 但调度器不触碰它。
