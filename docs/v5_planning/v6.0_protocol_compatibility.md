# v6.0 硅基体协议兼容性评估

**评估日期**: 2026-07-04  
**数据来源**: v4.0 真实模板 (F-005~F-040, 21 条模板)  
**状态**: 记录 only，不修改代码

---

## 1. 评估方法

基于 21 条真实模板逐字段分析，识别：
- **字段交集**: v2.0 模板 (F-005~F-020) 和 v4.0 新模板 (F-021~F-040) 的共同字段
- **结构差异**: 两种模板格式的不一致点
- **扩展兼容性**: 新产业模板能直接使用现有字段吗？需要新增哪些字段？

## 2. 字段一致性分析

### 2.1 现有字段分布

| 字段 | v2.0 通用 | v4.0 锂电 | v4.0 光伏 | 状态 |
|---|---|---|---|---|
| `fault_id` | ✅ | ✅ | ✅ | 完全一致 |
| `name` | ❌ 缺失 | ✅ | ✅ | v2.0 不兼容 |
| `category` | ❌ 缺失 | ✅ | ✅ | v2.0 不兼容 |
| `severity` | ✅ | ✅ | ✅ | 完全一致 |
| `description` | ✅ | ✅ | ✅ | 完全一致 |
| `signals` | ❌ 缺失 (含在 parameters) | ✅ list | ✅ list | v2.0 不兼容 |
| `affects` | ❌ 缺失 | ✅ list | ✅ list | v2.0 不兼容 |
| `repair_strategy` | ✅ | ✅ | ✅ | 完全一致 |
| `fault_type` | ✅ | ❌ 缺失 | ❌ 缺失 | v4.0 不兼容 |
| `parameters` | ✅ dict | ❌ 缺失 | ❌ 缺失 | v4.0 不兼容 |
| `source` | ✅ | ❌ 缺失 | ❌ 缺失 | v2.0 专有 |

**诊断**: 两代模板之间存在 **双向不兼容**:
- v2.0 缺 `name/category/signals/affects`
- v4.0 缺 `fault_type/parameters/source`

### 2.2 Category 分布 (仅 v4.0)

| Category | 数量 | 代表故障 |
|---|---|---|
| `process_anomaly` | 7 | 烧结温度、涂布增厚、焊带偏移 |
| `equipment_fault` | 5 | 辊压断带、分切毛刺、玻璃破裂 |
| `material_quality` | 4 | 浆料粘度、铁杂质、边框腐蚀 |
| `quality_defect` | 4 | EL 暗星团、IV 下降、接线盒粘接 |

**共 4 个 category，覆盖产线全流程。**

## 3. 跨产业适配性

### 3.1 现有策略能否覆盖新产业？

| 策略 | v2.0 (通用) | v4.0 (锂电) | v4.0 (光伏) | 结论 |
|---|---|---|---|---|
| S1 (文件补全/参数调整) | ✅ 12/16 (75%) | ✅ 7/10 (70%) | ✅ 4/10 (40%) | 锂电高度适配，光伏略低 |
| S2 (配置修复/质检判定) | ✅ 2/16 (13%) | ✅ 2/10 (20%) | ✅ 3/10 (30%) | 各产业均匀分布 |
| S3 (权限恢复) | ✅ 2/16 (13%) | ✅ 1/10 (10%) | ✅ 3/10 (30%) | 光伏安全相关场景更多 |

**核心结论**: 现有 S1/S2/S3 策略 **100% 可适配** 锂电和光伏产业。  
不需要新增修复策略。只需调整策略分布权重。

### 3.2 产业间字段差异

| 维度 | 锂电 | 光伏 | 统一潜力 |
|---|---|---|---|
| 核心故障触发 | 温度/粘度/压力信号 | EL 图像/IV 曲线/光学检测 | ✅ 可抽象为 `sensor_type` + `threshold` |
| 影响范围 | 批次/电芯 | 组件/串/二极管 | ⚠️ `affects` 字段格式不统一 |
| 严重等级标准 | P1/P2/P3 | P1/P2/P3 | ✅ 一致 |
| 修复权限 | 产线主管/质检 | 质检/电气工程师 | ⚠️ `rbac_level` 需标准化 |

## 4. 协议标准化建议 (v10.0 目标)

### 4.1 统一字段集 (proposed)

```yaml
# v10.0 建议标准格式
fault:
  # 基础 (所有版本统一)
  fault_id: str               # 唯一标识
  name: str                   # 故障名称
  description: str            # 描述
  severity: enum{P1,P2,P3}    # 严重等级

  # v5.0 标准化新增 (当前有差异)
  industry: enum{general,lithium,pv,energy_storage}
  category: str               # 故障分类 (process/material/equipment/quality)
  repair_strategy: enum{S1,S2,S3}

  # v10.0 扩展 (当前缺失)
  signals: list               # 结构化键值对信号
  affects: list               # 影响范围
  rbac_level: enum{L1,L2,L3}  # 修复权限等级
  sensor_type: str            # 信号来源传感器
  threshold: str              # 触发阈值表达式

  # 兼容字段 (v2.0 保留)
  fault_type: str             # 旧格式兼容
  parameters: dict            # 旧格式兼容
```

### 4.2 格式迁移策略

| 步骤 | 操作 | 影响 |
|---|---|---|
| 1 | v4.0+ 模板统一用 `templates` list 格式 | 不影响 v2.0 单对象格式 |
| 2 | v5.0 调度器读取时自动补齐 v2.0 缺失字段 | 兼容读取 |
| 3 | v6.0 新建模板强制用 v6.0 标准格式 | 不强制迁移旧模板 |
| 4 | v10.0 全量模板标准化 | 支持自动迁移脚本 |

## 5. 结论

| 维度 | 状态 | 说明 |
|---|---|---|
| 跨产业适配 | ✅ 当前策略 100% 可覆盖锂电/光伏 | 不需新增策略 |
| 字段兼容性 | ⚠️ v2.0 和 v4.0 格式略有不一致 | 自动补齐即可 |
| 协议标准化 | 📌 建议 v10.0 统一 | 当前不需要做 |
| 兼容风险评估 | 🟢 低 | v2.0 读兼容已有 mes_adapter.py 实现 |
